# 课题组轮转任务清单：线性层编码验证与参数配置

本项目涉及 Cachemir 架构与全同态加密 (FHE) 技术中线性层代码的验证。为适应轮转实习周期，本阶段任务聚焦于两个核心目标：验证现有架构下线性层编码的准确性，以及实现模型参数外置配置，并解决任意维度参数下的编码对齐问题。

请按照以下指导分工推进研究与开发工作：

---

## 阶段一：线性层正确性验证 (Encoding Correctness Validation)
**任务背景**：在修改内部参数维度前，需先基于当前有效的系统参数，建立工程测试框架，量化评估线性层明文代码与密文代码在计算结果上的误差范围。

* [ ] **任务 1.1: 搭建自动化对比测试框架**
  * 在项目代码库中新增独立的测试文件（如 `linear_test.go`）。
  * 流程：针对给定的明文输入 `x`，分别调用现有的明文路线 `LinearMsg()` 和同态计算路线 `Linear()` 进行推断。对 `Linear()` 模块的输出结果执行解密与解码操作。

* [ ] **任务 1.2: 执行误差评估与断言**
  * 在上述框架中，加入均方误差（MSE）计算函数。
  * 将“明文路线执行结果”与“密文路线解密结果”进行逐元素比对。设置合理的误差容忍范围（如 1e-4）用于自动化断言。
  * 选用目前代码中支持的常规参数组合，遍历核心算子：`QKV` 映射、`Out`、`UpGate` 以及 `Down` 层。整理并记录各类算子的初始 MSE 误差作为后期阶段的功能正确性参考基线。

---

## 阶段二：任意维度适配与参数配置化 (Parameter Configuration & Dimension Alignment)
**任务背景**：当前 `main.go` 与 `util.go` 等文件中存在固定的维度假设（例如 `math.Sqrt()` 的完全平方数限制）。输入非常规的尺寸参数时，会造成底层同态打包机制无法对齐并引发程序退出。该阶段旨在重构参数配置并解决计算维度的输入对齐问题。

* [ ] **任务 2.1: 提取参数配置模块**
  * 引入诸如 JSON 或 YAML 的标准解析库，接管代码中对 `seqLen`, `numHeads`, `hidDim`, `expDim` 等变量的初始化流程。
  * 使得用户可以通过修改一份外部配置文件，初始化整个运行环境的 `LlamaSize` 结构体，而无需在源码中调整 `flag` 参数。

* [ ] **任务 2.2: 解决底层计算维度的对齐兼容限制**
  * **需求**：排查修改使得各种网络参数形状（例如不对称的行列尺寸或质数参数）通过 `Linear()` 模块时不引发错误。
  * **实施建议（可根据难度自主选择）**：
    1. 方法一：前置数据处理封装。在真实数据交付底层算子前，将其矩阵维度补齐至底层函数所约束的最近规整尺寸，结算后再加以截除截断还原。
    2. 方法二：重构底层机制。研究底层由 CKKS 进行旋转与打包的计算原理，重新梳理并修改 `util.go` 及 `linear.go` 中的分块或对齐规则逻辑以支持任意大小参数。

* [ ] **任务 2.3: 完成极限参数测试并撰写实验日志**
  * 在任务 2.1 与 2.2 结束后，挑选结构极端的尺寸集（不可被正常规整因数分解的长宽尺寸）。
  * 将其重新传入任务 1.1 完成的对比框架中进行运算，得出各线性模块的 MSE 数据。
  * 同步输出带有上述相关参数特征及 MSE 误差比对结果的结题文档日志。
