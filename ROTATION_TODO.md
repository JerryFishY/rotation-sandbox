# 课题组轮转任务清单：线性层正确性验证与参数配置动态化

本项目涉及 Cachemir 架构与全同态加密 (FHE) 技术中线性层代码的验证。为适应轮转实习的紧张周期，本阶段任务聚焦于两个核心目标：验证现有架构下线性层编码的准确性，以及实现模型参数外置配置，并解决任意维度参数下的编码对齐问题。

请按照以下指导分工推进研究与开发工作：

---

## 零、环境跑通与开发首选项 (Setup & AI Coding)
在正式开发前，请优先搭建本地运行环境，并采取现代化的研发工作流！

* [ ] **本地初始测试运行**
  * 确保已经安装 Go 语言环境 (版本需在 1.20 以上可以支持泛型)。
  * 进入项目根目录执行 `go mod tidy` 拉取依赖库（本例核心为 Lattigo）。
  * 尝试运行现有参数来验证环境已通，例如：
    `go run main.go -test="Decoder" -hidDim=32 -seqLen=29`

* [ ] 💡 **学长的高优建议：必须全程用好 AI 写代码！**

  * **行动指南**：
    1. 强烈建议使用搭载了 AI 代码补全的 IDE (如 Cursor, Windsurf)，或者在网页端配合使用 Claude / ChatGPT。
    2. 当你需要写“读取 JSON 配置文件”、“生成均方误差数组对比函数”、甚至是“遇到 panic 数组越界的报错”时，**请直接将包含这些错误的上下游原始代码片段复制喂给 AI，并让它给出正确的 Go 代码**，以此作为你代码落地的底座。
    3. 把大模型作为你经验丰富的“副驾驶”，在它的辅助下主攻下述的核验难点。

## 阶段一：线性层正确性验证 (Encoding Correctness Validation)
**任务背景**：在修改内部参数维度前，需先基于当前有效的系统参数，建立工程测试框架，量化评估线性层明文代码与密文代码在计算结果上的误差范围。

* [ ] **任务 1.1: 搭建自动化对比测试框架**
  * 在项目代码库中新增独立的测试文件（如 `linear_test.go`）。
  * 流程：针对给定的明文输入 `x`，分别调用现有的常规路线 `LinearMsg()` 和同态计算路线 `Linear()` 模块运行。并对 `Linear()` 模块的输出结果执行解密还原操作。

* [ ] **任务 1.2: 执行误差评估与框架断言**
  * 在上述框架中，加入均方误差（MSE）计算函数。
  * 将“明文路线执行结果”与“密文路线解密结果”进行逐元素比对。设置合理的误差容忍范围（如 1e-4）用于自动化断言。
  * 选用目前代码中天生就可以被整除跑通的常规参数组合，遍历核心算子：`QKV`, `Out`, `UpGate`, 以及 `Down`。记录各类算子的初始误差作为接下来阶段的功能参考基线。

## 阶段二：任意维度适配与参数配置化 (Parameter Configuration & Dimension Alignment)
**任务背景**：系统代码中的同态多项式设计存在固定的维度约束（例如限制参数必须由某些能开平方的维度衍生而来）。如果硬性传入非常规的尺寸参数，则底层打包会因为不对齐而报错崩溃。本阶段要彻底解决该参数断言缺陷。

* [ ] **任务 2.1: 提取参数配置模块**
  * 引入或令 AI 辅助编写 JSON/YAML 解析器，接管代码中对 `seqLen`, `numHeads`, `hidDim`, `expDim` 的初始化。
  * 达成只使用外部一份配置，即可随时无缝驱动、设定环境参数结构体。

* [ ] **任务 2.2: 攻克底层的维度对齐兼容限制**
  * **需求**：排查并更新系统，使得当输入一些不对称元素（包括素数组成的矩阵大小长宽）去过核心计算逻辑 `Linear()` 模块时不出错。
  * **实施建议（均可以和 AI 结对编程摸索）**：
    1. *路线一（绕路围堵法）*：将重点放在真实的载体数据本身。在送交付算子之前补齐维度包裹，然后在最后的数据解码阶段执行截断以过滤补零冗余元素，靠输入输出来完成间接处理。
    2. *路线二（内核切除法）*：研究 CKKS 的打包算法，分析代码中 `math.Sqrt()` 的硬性条件如何运作，随后重构 `util.go` 及 `linear.go` 的切块分配方法以提供原生不挑尺寸的数据消化能力。

* [ ] **任务 2.3: 完成极限参数测试并撰写实验日志**
  * 将极限奇偶性且没有显性公约数的尺寸集当做环境配置。
  * 将其重新传回阶段一写好的验证中跑出 MSE 数据。
  * 输出囊括验证情况相关参数集特征及 MSE 的实验结题日志评估报告。
